================================================================================
SECTION 1: Publicly captured endpoint shapes and normalization/trade-planning fields
================================================================================

Endpoint: GET /markets (gamma-api.polymarket.com)
Auth: none
Returns: JSON array of market objects

--- market: identification ---
  id                   string    unique market identifier
  question             string    bet question text
  slug                 string    URL slug
  conditionId          string    on-chain condition ID (CTF contract)
  questionID           string    question identifier
  description          string    resolution criteria

--- market: outcomes/pricing ---
  outcomes             string    JSON array of labels, e.g. '["Yes","No"]'
  outcomePrices        string    JSON array of prices, e.g. '["0.65","0.35"]'
  clobTokenIds         string    JSON array of CLOB token IDs (needed for orders)
  bestBid              number    best bid price
  bestAsk              number    best ask price
  spread               number    bid-ask spread
  lastTradePrice       number    last trade price

--- market: availability (explicit semantics) ---
  active               boolean   market is active
  closed               boolean   market has resolved
  archived             boolean   market is archived
  acceptingOrders      boolean   CLOB accepting orders (tradeable now)
  endDate              string    resolution deadline (ISO 8601)
  Availability rule: active=true AND closed=false -> live market
                     acceptingOrders=true -> can place orders
                     acceptingOrders=false -> view-only

--- market: order book config ---
  enableOrderBook      boolean   CLOB enabled
  orderPriceMinTickSize number   min price tick (e.g. 0.01)
  orderMinSize         number    min order size
  negRisk              boolean   multi-outcome negative risk market
  feeType              string    fee type
  feesEnabled          boolean   fees enabled

--- market: volume/liquidity ---
  volume, volumeNum, volume24hr
  liquidity, liquidityNum, liquidityClob

Endpoint: GET /events (gamma-api.polymarket.com)
  Events group multiple markets. Include: id, title, ticker, slug,
  tags[{id,label,slug}], markets[], active, closed, volume, openInterest

Endpoint: GET /tags (gamma-api.polymarket.com)
  Tag objects: {id, label, slug}

Endpoint: GET https://data-api.polymarket.com/positions?user={address}
  Public, no auth. Returns JSON array of user positions.
  Example artifact: polymarket/positions-example.json (empty result for zero address)

Public CLOB read endpoints (https://clob.polymarket.com, no auth):
  GET /book?token_id={tokenID}       order book
  GET /prices?token_id={tokenID}      current prices
  GET /midpoint?token_id={tokenID}    midpoint price

SECTION 2: Additional fields required for trade execution and position-view
================================================================================

CLOB order placement (sourced from polymarket/py-clob-readme.md):

  Auth headers (from clob-client docs):
    POLY_ADDRESS        wallet address
    POLY_API_KEY        API key (derived via createOrDeriveApiKey)
    POLY_PASSPHRASE     API passphrase
    POLY_SIGNATURE      request signature
    POLY_TIMESTAMP      request timestamp

  Limit order (from py-clob-client OrderArgs):
    token_id            string    CLOB token ID from market.clobTokenIds
    price               float     price 0.00-1.00
    size                float     number of shares
    side                string    BUY or SELL
  OrderType: GTC (good-til-cancelled)

  Market order (from py-clob-client MarketOrderArgs):
    token_id            string    CLOB token ID
    amount              float     USDC amount
    side                string    BUY or SELL
  OrderType: FOK (fill-or-kill)

  Underlying POST /order body (inferred from SDK source code, not directly from tracked raw REST docs):
    order: {
      maker             address   maker wallet
      signer            address   signer address
      taker             address   taker (0x0 for open)
      tokenId           string    CTF token ID
      makerAmount       string    maker amount
      takerAmount       string    taker amount
      side              string    BUY or SELL
      expiration        string    order expiration timestamp
      nonce             string    unique nonce
      feeRateBps        string    fee rate in basis points
      signature         string    EIP-712 signature
      salt              string    order salt
      signatureType     integer   0=EOA, 1=Magic/Email
    }
    owner               address   order owner
    orderType            string    GTC, FOK, GTD

  ClobClient initialization (from py-clob-readme.md):
    host = "https://clob.polymarket.com"
    chain_id = 137  (Polygon)
    key = PRIVATE_KEY
    signature_type = 0 (EOA) or 1 (Magic/Email)
    funder = FUNDER_ADDRESS

  Position view:
    Public: GET https://data-api.polymarket.com/positions?user={address}
    Auth:   GET https://clob.polymarket.com/orders?owner={address} (API key required)

================================================================================
SECTION 3: Discoverability of execution/position fields
================================================================================

Publicly discoverable (no auth):
  - All market metadata: id, question, outcomes, prices, volume, liquidity
  - CLOB token IDs (from market.clobTokenIds)
  - Order book: GET /book?token_id={id}
  - Prices: GET /prices?token_id={id}
  - Positions: GET https://data-api.polymarket.com/positions?user={address}
  - Availability: active, closed, archived, acceptingOrders
  - Order config: tick size, min size, negRisk
  - Events with tags (categories)

Publicly sourced in this artifact package:
  - CLOB client contract: polymarket/py-clob-readme.md, polymarket/clob-client-readme.md
  - Positions endpoint example: polymarket/positions-example.json
  - POST /order body shape (inferred from SDK source, not from tracked raw REST docs)

Wallet/user-specific (requires API key + wallet signature):
  - Place/cancel orders (EIP-712 signature required)
  - View own open orders (via CLOB API with auth headers)
  - API key derived via createOrDeriveApiKey with wallet

Not publicly sourced in this artifact package:
  - Exact fee schedule (bps rates not in market object or CLOB docs)
  - Reward calculation details
  - Resolution oracle internals
