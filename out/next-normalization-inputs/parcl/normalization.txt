SECTION 1: Publicly captured endpoint shapes and normalization/trade-planning fields

Endpoint: GET /v1/exchange
Auth: none
Returns: JSON object with exchange-level config

--- exchange ---
  address              string    exchange program address
  id                   string    exchange ID
  collateral_mint      string    USDC mint address
  collateral_expo      integer   collateral exponent (-6 for USDC)
  status               integer   exchange status (0 = active)
  market_ids           int[]     list of active on-chain market IDs

--- exchange.settings ---
  min_lp_duration      integer   minimum LP duration (seconds, 2592000 = 30 days)
  settlement_delay     integer   settlement delay (seconds, 86400 = 24 hours)
  min_liquidation_fee  string    minimum liquidation fee (500000 = 0.50 USDC)
  max_liquidation_fee  string    maximum liquidation fee
  protocol_fee_rate    integer   protocol fee rate (2000 = 0.20%)
  locked_open_interest_ratio integer locked OI ratio (1500)
  max_keeper_tip_rate  integer   max keeper tip rate

--- exchange.accounting ---
  balance              string    total exchange balance
  margin_balance       string    total margin balance
  lp_balance           string    LP balance
  notional_open_interest string  total notional OI

Endpoint: GET /v1/market-ids
Auth: none
Returns: JSON { "market_id": "solana_address", ... } mapping IDs to on-chain addresses

Endpoint: GET /v1/market?market_id={id}
Auth: none
Returns: JSON object with per-market config

--- market ---
  address              string    market program address
  id                   string    market ID (matches exchange.market_ids)
  exchange             string    parent exchange address
  price_feed           string    Pyth price feed address
  status               integer   market status

--- market.price_feed_info ---
  price                string    current price (scaled by expo, e.g. "57565000000" at expo -8 = $575.65/sqft)
  expo                 integer   price exponent (typically -8)

--- market.settings (margin/leverage/fee config per market) ---
  initial_margin_ratio         integer   initial margin ratio (20000 = 5x max leverage)
  min_initial_margin_ratio     integer   min initial margin ratio (500 = 200x theoretical cap)
  maintenance_margin_proportion integer  maintenance margin proportion (8000 = 80%)
  maker_fee_rate               integer   maker fee (40 = 0.004%)
  taker_fee_rate               integer   taker fee (60 = 0.006%)
  liquidation_fee_rate         integer   liquidation fee rate
  min_position_margin          string    minimum position margin (500000 = 0.50 USDC)
  max_side_size                string    max size per side
  skew_scale                   string    skew scale for funding
  max_funding_velocity         integer   max funding velocity
  max_liquidation_pd           integer   max liquidation price delta
  max_liquidation_limit_accumulation_multiplier integer

--- market.accounting ---
  size                 string    total market size
  skew                 string    current skew (long - short)
  last_funding_rate    string    last funding rate
  last_funding_per_unit string   last funding per unit

SECTION 2: Additional fields required for trade execution and position-view
================================================================================

Parcl v3 API (https://v3.parcl-api.com):

Create margin account:
  POST /v1/create-margin-account-transaction
  {
    "exchange":         string    exchange address
    "owner":            string    wallet public key
  }
  Returns: serialized Solana transaction to sign and submit

Deposit margin:
  POST /v1/deposit-margin-transaction
  {
    "margin_account":   string    margin account address
    "amount":           string    USDC amount to deposit (scaled by collateral_expo)
  }

Open/modify position:
  POST /v1/modify-position-transaction
  {
    "margin_account":   string    margin account address
    "market_id":        integer   market ID from exchange.market_ids
    "size_delta":       string    position size change (positive = long, negative = short)
    "acceptable_price": string    slippage-protected price
  }

Get modify-position quote (pre-trade):
  POST /v1/modify-position-quote
  {
    "margin_account":   string    margin account address
    "market_id":        integer   market ID
    "size_delta":       string    proposed size change
  }
  Returns: estimated fill price, fees, margin impact

Close position:
  POST /v1/modify-position-transaction with opposing size_delta

Close margin account:
  POST /v1/close-margin-account-transaction
  { "margin_account": string }

View margin account:
  GET /v1/margin-account?margin_account={address}
  Returns: positions, collateral, margin health

View multiple margin accounts:
  POST /v1/margin-accounts
  { "margin_accounts": [address, ...] }

Liquidate (keeper):
  POST /v1/liquidate-transaction
  { "margin_account": string, "market_id": integer }

View unhealthy accounts:
  GET /v1/unhealthy-margin-accounts

================================================================================
SECTION 3: Discoverability of execution/position fields
================================================================================

Publicly discoverable (no auth, no wallet):
  - All on-chain market IDs and Solana addresses
  - Per-market margin ratios, fee rates, size limits, skew scales
  - Exchange-level settings (protocol fee, settlement delay, LP duration)
  - Current prices via Pyth oracle (price_feed_info)
  - Current funding rate and skew per market
  - Total exchange balance, OI, margin balance
  - Unhealthy margin accounts (GET /v1/unhealthy-margin-accounts)
  - Market names (via app.parcl.co __NEXT_DATA__)

Wallet/user-specific (requires margin account address, derived from wallet):
  - Margin account state: positions, collateral, unrealized P&L
  - Position details per market: size, entry price, funding accrued
  - Margin health / liquidation proximity
  - Modify-position quotes (requires margin_account param)
  - All write operations (open/close/deposit/withdraw) require wallet signature

Not publicly discoverable from these endpoints:
  - Historical price data per market (available via Parcl Labs data API, separate auth)
  - Trade history per user
  - Exact liquidation price (derivable from margin account state + market settings)
  - Future market additions (upcoming markets have no on-chain config yet)
