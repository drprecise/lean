================================================================================
SECTION 1: Publicly captured endpoint shapes and normalization/trade-planning fields
================================================================================

Endpoint: GET https://data.api.drift.trade/stats/markets
Auth: none
Returns: JSON { success: bool, markets: [] }

--- markets[] (one per market) ---
  symbol            string    e.g. "SOL-PERP", "USDC"
  marketIndex       integer   on-chain market index (required for all operations)
  marketType        string    "perp" or "spot"
  uiStatus          string    "visible", "hidden", or null
  baseAsset         string    base asset symbol
  quoteAsset        string    quote asset symbol (USDC for perps)
  status            string    "active", "settlement", "reduceOnly"
  precision         integer   price precision (decimal places)

--- markets[].limits ---
  leverage.min      number    minimum leverage (perps only, e.g. 1)
  leverage.max      number    maximum leverage (perps only, e.g. 20)
  amount.min        number    minimum position size in base asset
  amount.max        number    maximum position size in base asset

--- markets[].fees ---
  maker             number    maker fee rate (negative = rebate, e.g. -0.00002)
  taker             number    taker fee rate (e.g. 0.00035)

--- markets[] price/volume (real-time public state) ---
  oraclePrice       string    current oracle price
  markPrice         string    current mark price (perps; includes funding)
  price             string    current price
  priceChange24h    string    24h price change
  priceChange24hPercent string 24h price change %
  priceHigh         string    24h high
  priceLow          string    24h low
  baseVolume        string    24h base volume
  quoteVolume       string    24h quote volume
  openInterest      string    current open interest (perps)
  fundingRate       string    current funding rate (perps)
  fundingRateUpdateTs string  funding rate last update timestamp

Leverage/margin summary (from limits, perps only):
  SOL-PERP    1-20x     BTC-PERP    1-20x     ETH-PERP    1-20x
  Most perps  1-20x     Some perps  1-10x or 1-5x (smaller markets)

Fee summary:
  Perp maker: -0.00002 (rebate)     Perp taker: 0.00035
  Spot maker: -0.00002 (rebate)     Spot taker: 0.00035
  (uniform across all markets in current data)

================================================================================
SECTION 2: Additional fields required for trade execution and position-view
================================================================================

Drift Gateway API (self-hosted or via drift-gateway binary):
  Base: http://localhost:8080 (local) or custom deployment

Open perp position:
  POST /v2/orders
  {
    "marketIndex":    integer   market index from stats/markets
    "marketType":     string    "perp"
    "amount":         string    base amount (negative = short)
    "price":          string    limit price (0 for market)
    "orderType":      string    "market", "limit", "triggerMarket", "triggerLimit"
    "reduceOnly":     bool      reduce-only flag
    "userOrderId":    integer   optional client order ID
  }

Open spot order:
  POST /v2/orders
  {
    "marketIndex":    integer   market index
    "marketType":     string    "spot"
    "amount":         string    base amount (negative = sell)
    "price":          string    limit price
    "orderType":      string    "market" or "limit"
  }

Cancel order:
  DELETE /v2/orders
  { "marketIndex": int, "marketType": string, "userOrderId": int }

View positions:
  GET /v2/positions (requires authority/wallet in gateway config)

View orders:
  GET /v2/orders (requires authority/wallet in gateway config)

Direct SDK (drift-sdk-ts / drift-sdk-py):
  Requires: Solana wallet keypair, RPC endpoint
  Methods: placeOrder, cancelOrder, getUser, getUserAccount

================================================================================
SECTION 3: Discoverability of execution/position fields
================================================================================

Publicly discoverable (from stats/markets, no auth):
  - All market definitions with symbol, index, type, status
  - Leverage ranges per market (min/max)
  - Fee rates (maker/taker) per market
  - Real-time oracle/mark prices, funding rates
  - Open interest, volume, price changes
  - Amount limits (min/max position size)

Wallet/user-specific (requires Solana keypair or gateway auth):
  - Open positions and their entry prices, P&L, liquidation prices
  - Open orders (limit, trigger)
  - Account health, margin ratio, free collateral
  - Historical fills and settlements
  - Deposit/withdraw collateral operations

Not publicly discoverable from this endpoint:
  - Per-user fee tier (based on 30d volume; tiers exist but user tier requires account lookup)
  - Insurance fund state, revenue pool state
  - Exact liquidation price (derivable from position + oracle + margin params)
  - Historical funding rate payments per user
